- **`commands` adapter (`ChannelCommandAdapter`)** drives the runtime's slash-command menu system (`/commands`, `/models`, `/model <provider/model>`). It is exported from `openclaw/plugin-sdk/channel-runtime` (NOT `channel-core`/`channel-contract`) and lives on `base` (forwarded by `createChatChannelPlugin`'s `{ ...params.base }`). Two halves: (1) auto-enable flags `nativeCommandsAutoEnabled`/`nativeSkillsAutoEnabled` (default the channel ON for slash commands + skills; without them commands exist but stay off until manually enabled); (2) `build*ChannelData` builders render native interactive elements for the command/menu replies — Telegram returns `{ telegram: { buttons: inlineKeyboardRows } }`, Discord returns component containers. For a plugin channel the channelData just needs to be a non-empty object (`hasReplyChannelData` is `Object.keys(value).length > 0`) so the runtime routes the reply through `outbound.sendPayload` instead of `sendText`. The Cliq bridge is to return the SAME `{ cliqCard: { buttons } }` marker that `outbound-presentation.sendPayload` already reads — zero extra wiring, the buttons ship via `CliqClient.sendCard`. Cliq's callback-button analog is `{ action: "invoke", data: <slashCommandText> }`: tapping it posts `data` back to the Deluge handler, which forwards it as an inbound message whose text IS `data`, so each button's `data` is the slash command to re-parse (`/models openai`, `/model openai/gpt-4`, `/models openai 2` for pagination). `/commands` pagination is a bundled-channel callback-edit-in-place feature the runtime's `/commands` handler does NOT accept a page arg (exact-match only, `commands-handlers.runtime-DmKG7VZH.js:4145`), so a plugin channel's `buildCommandsListChannelData` returns `null` and the commands list degrades to plain text (the correct UX). `/models <provider> <page>` IS supported (`parseModelsArgs` → `parseListArgs` treats a bare numeric token as the page), so model-list prev/next buttons can paginate. Cliq's 10-button cap means the nav row (up to 4: prev + page-indicator + next + back) must be computed FIRST and the model grid fills the remaining slots, never the other way around.
