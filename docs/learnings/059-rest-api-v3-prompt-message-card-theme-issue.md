- **REST API v3 `prompt` Message Card theme (issue #63).** The `prompt` theme is a focused quick-reply card: just a `title` (the question / alert text, ≤200 chars — same first-line split as `modern-inline`) and 1–5 action `buttons`. Per the v3 Message Cards docs (`Available Themes` table), `prompt`'s REQUIRED fields are `theme`, `title`, `buttons` (min 1, max 5) — it has NO `sections` / `thumbnail` (those are `modern-inline`-only). The button shape is the SAME as `modern-inline` (`{ label, action: { type, data } }`, same v2→v3 action mapping). The distinguishing contract: a `prompt` with no buttons is INVALID, so `cliqCardToV3MessageCard({ theme: "prompt" })` returns `null` when no buttons survive conversion (rather than emitting a buttonless card the API would reject); the caller then falls back to v2 / plain text. The top-level `text` fallback and `slides` (a `text` slide for the body remainder) apply to `prompt` exactly as they do to `modern-inline` — `slides` is theme-independent per the docs. The `theme` is selected via a new optional `theme` field on `SendCardMessageOptions` / `CliqRenderedCard` (`"modern-inline"` default — preserves every existing call site). The plugin's quick-reply command buttons (`src/commands.ts`: `/models`, `/model` menus — each button posts a slash command back to the bot via `action: "invoke"`) set `theme: "prompt"` on their `cliqCard` channel-data marker; the v2 path ignores the field and keeps the raw top-level `buttons` array, so v2 behavior is unchanged. `prompt` reuses the SAME OAuth scopes as `modern-inline` (`Channels.CREATE` for channel cards, `Webhooks.CREATE` for DM cards) — no new scope.
