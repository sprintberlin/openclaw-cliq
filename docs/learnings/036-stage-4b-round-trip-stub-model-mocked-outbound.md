- **Stage-4b round-trip: stub model + mocked outbound via `apiBase`/`oauthBase` config overrides.** The hard-coded EU endpoints (`https://cliq.zoho.eu` / `https://accounts.zoho.eu`) prevent the smoke from intercepting the outbound OAuth + send path, so the channel config exposes optional `apiBase`/`oauthBase` overrides (single-account at `channels.cliq.*` AND per-account at `channels.cliq.accounts.<id>.*` via the shallow-merge resolution). `CliqClientRegistry.getOrCreate` threads `account.apiBase`/`account.oauthBase` into the `CliqClient` constructor (which already had `apiBase`/`oauthBase` params defaulting to the EU URLs). This is also a production-useful feature for self-hosted / alternate Zoho data centers. The Stage-4b smoke (`scripts/stage4b-mock.mjs` + the `[9/12]`–`[12/12]` section of `smoke-gateway.sh`) starts a single loopback HTTP mock that stands in for THREE upstreams at once: Zoho OAuth (`POST /oauth/v2/token`), the Cliq bot-message send (`POST /api/v2/bots/{botId}/message`), and an OpenAI-compatible stub chat model (`POST /v1/chat/completions`). The stub model is registered as a custom provider `models.providers.stub = { baseUrl: "<mock>/v1", apiKey: "stub-key", models: [{ id: "echo", api: "openai-completions" }] }` with `agents.defaults.model = "stub/echo"`; the `openai-completions` API hits `<baseUrl>/chat/completions` (i.e. `<mock>/v1/chat/completions`). The stub echoes the latest user message back as `stub-reply: <userText>`, so the smoke asserts the round-trip by polling the mock's sends log (a JSONL file the mock appends to on each bot send) for a line containing both `stub-reply:` and the unique inbound marker. With the default `ackPolicy: "after_dispatch"`, the HTTP 200 returns once dispatch resolves, but the outbound delivery (agent reply → OAuth → bot send) happens AFTER the 200, so the sends-log poll must allow ~40s of slack. Stage-4b reuses the SAME profile as Stage-4 (it kills the Stage-4 gateway, rewrites `channels.cliq` with the mock `apiBase`/`oauthBase` + the `models`/`agents` blocks, and restarts the gateway); the throwaway `HOME`/`OPENCLAW_STATE_DIR`/`OPENCLAW_CONFIG_PATH` isolation from Stage-4 carries over.
