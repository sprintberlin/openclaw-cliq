- **REST API v3 `poll` Message Card theme (issue #64).** The `poll` theme is a voting card: a `title` (the poll question, ≤200 chars — same first-line split as the other themes) plus 2–10 `options` (each `{ text }`, ≤100 chars). Per the v3 Message Cards docs (`Available Themes` table), `poll`'s REQUIRED fields are `theme`, `title`, `options` (min 2, max 10) — it has NO `buttons` / `sections` / `thumbnail` (those are `modern-inline`/`prompt`-only). The CRITICAL difference from `modern-inline`/`prompt`: a poll's `options` are NOT action buttons. Cliq tracks live vote counts + percentages **natively** — a vote does NOT post anything back to the bot (votes are counted in-place by Cliq, not surfaced as an inbound message via `invoke.bot`), so poll options carry only a `text` field (no `action`/`data`/`url`). Consequence: the poll input shape needs a NEW `pollOptions: string[]` field on `CliqV3CardInput` / `SendCardMessageOptions` / `CliqRenderedCard` (NOT a reuse of `buttons`), and `cliqCardToV3MessageCard` IGNORES `buttons` for `theme === "poll"` (a poll has no action buttons). `options` is REQUIRED (min 2), so the renderer returns `null` when fewer than 2 options survive (empties / whitespace dropped before counting; options capped at `V3_MAX_POLL_OPTIONS = 10`, over-length clamped to `V3_MAX_POLL_OPTION_LENGTH = 100` chars with an ellipsis — same clamp pattern as button labels but a different limit). The top-level `text` fallback and `slides` (a `text` slide for the body remainder) apply to `poll` exactly as for the other themes. The agent-facing surface is the shared `message` tool: `message(action=send, theme="poll", pollOptions=["A","B",...])` posts a poll (the `message` text is the poll question); the `message` tool schema is `null` (params flow through regardless of the schema fragment, so the new `theme` + `pollOptions` params are documented in the agent prompt hints instead of a schema). `poll` reuses the SAME OAuth scopes as the other card themes (`Channels.CREATE` for channel cards, `Webhooks.CREATE` for DM cards) — no new scope. With `poll` rendered, all THREE documented v3 Message Card themes are complete; the only remaining v3 Message Card surface is `slides` beyond the `text` slide (table / list / label / images supporting content — a Phase 3 item).
